# ADR 0001: Popup 作为控制面板而非文本翻译器

## 状态
Accepted

## 背景
当前 popup 提供了一个文本输入框，让用户手动输入或粘贴文本进行单条翻译。这一功能与页面内的**划词翻译**高度重叠：用户完全可以在网页上选中文字后翻译，而不需要打开 popup。同时 popup 的生命周期较短，不适合承载长文本输入和复杂的编辑操作。

## 决策
将 popup 重构为一个**小巧、精巧的控制面板**，核心目标从“输入并翻译”转向“查看状态并快速触发”。具体包括：

- **移除逐句输入翻译**：取消大段 textarea 和“立即翻译”主按钮，避免与划词翻译功能重复。
- **提供整页翻译入口**：放置显眼的“翻译整页”按钮，点击后向当前标签页 content script 发送翻译指令。
- **流式翻译开关**：提供全局流式模式开关，控制 content script 中划词/整页翻译是否启用 SSE 实时输出。
- **用量看板**：显示本次会话/累计 token 用量（超过 1,000 按 `k` 显示）、缓存命中率、词库/热词数量。
- **模型切换下拉栏**：从已保存的 ProviderProfile 列表读取，允许在 popup 中快速切换当前使用的供应商与模型。
- **连接状态指示**：popup 打开时异步检测当前 provider 是否可连接，用红黄绿三色状态标表示。

## 后果

- 需要新增 popup → content script 的整页翻译消息通道。
- background 需要维护 token 用量统计（由响应中 usage 字段或字符数估算）。
- 模型下拉栏的数据源改为 ProviderProfile 列表，要求 Options 页先完成档案化改造。
- 视觉上保持现代简约风格，宽度仍控制在 320px，信息密度提高但需避免拥挤。

## 拒绝的替代方案

- **保留输入框但缩小**：与划词翻译重叠的问题未解决，且小输入框体验更差。
- **在 popup 内做完整翻译历史**：会增加存储和 UI 复杂度，偏离“轻巧控制面板”定位。
