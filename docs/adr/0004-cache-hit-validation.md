# ADR 0004: 缓存命中质量校验机制

## 状态

Accepted

## 背景

`TranslationCache` 采用 `sourceLang:targetLang:style:text` 键匹配。初期通过较宽松的文本归一化（引号、破折号、全角半角统一）和短词缓存来提升命中率，但带来了**近似命中**问题：例如短词 `"github"` 被缓存为某仓库标题，后续任意 `"github"` 都会命中这条错误译文。

因此把策略收紧为「只保留语义完全一致的缓存命中输出」：弱化键归一化、提高源文长度门槛、清空内置短词热词库，并在命中返回前增加质量校验，拦截张冠李戴、模型拒绝文本、回显原文等坏命中。

## 决策

在缓存读取和写入两层加入**严格准入 + 四层短路校验器**，并在扩展启动时异步清理存量无效条目。

### 规则优先级（短路执行）

| 优先级 | 规则 | 目的 | 实现要点 |
|---|---|---|---|
| 准入 | **源文长度下限** | 短词语义歧义大，易产生近似命中 | 归一化后源文长度 `< 12` 的条目不缓存、不命中 |
| 0 | **拒绝/错误文本拦截** | 拦住模型没翻译而返回的拒绝、错误、HTML 片段 | `I'm sorry`、`as an AI`、`cannot translate`、`<!DOCTYPE`、`<html`、`429`、`rate limit` 等子串匹配 |
| 1 | **短源文长度/比例（语言对敏感）** | 防止短词被配超长幻觉译文 | 源文 ≤10 字符时触发；CJK→拉丁 5x、拉丁→CJK 2x、同语系 3x |
| 2 | **跨语种回显原文** | 防止模型把原文原样返回却被缓存 | 源语 ≠ 目标语且归一化后译文 == 源文时拦截；专有名词/品牌词白名单例外 |
| 3 | **目标语合法性** | 防止译文语言方向错误 | 非 CJK→CJK：目标语脚本占比 >50%，且译文语言不能等于源语；CJK→CJK：仅检查不出现拉丁字符 |
| 4 | **短源文实体漂移** | 防止短词译文里凭空出现 URL、仓库名、命名实体 | 仅对 ≤10 字符源文检查 |

### 关键设计取舍

- **最小化键归一化**：只保留 NFC、去除零宽字符、折叠空白；引号、破折号、省略号、全角半角差异视为不同文本，避免不同源文因归一化折叠成同一个近似键。
- **源文长度下限**：低于 12 字符的源文存在较大语义歧义，直接不进入缓存，从源头减少近似命中。
- **清空内置热词库**：`BUILTIN_CACHE` 置空，不再预加载短词/固定译法，缓存只保留用户真实翻译且通过校验的条目。
- **读写双过滤**：写入前拦截避免制造新垃圾；读取前再校验可处理历史脏数据。
- **CJK→CJK 弱化脚本检测**：中/日/韩共享 CJK Unicode 区块，脚本占比无法区分语种，因此该语言对下只检查拉丁字符漂移，主要依赖规则 2（回显检测）。
- **规则 3 采样**：译文 >200 字符时，取前/中/后各 100 字符采样统计，降低长文本下的 CPU 开销。
- **专有名词白名单 + 启发式识别**：内置常见技术/品牌词，并自动识别首字母大写或全大写的单/多词短语，减少回显规则误伤。
- **启动清理**：Service Worker 初始化后异步扫描内存缓存，移除无效条目并同步到 IndexedDB。

### 指标

新增 `badHitRate = (blockedHits + userReportedHits) / (cacheHits + blockedHits)`，并维护 `blockedByRule` 分类计数，用于后续调优阈值。

## 后果

- 缓存命中率会明显下降，因为短词和近似变体不再命中；但命中质量显著提升，几乎只保留语义一致的句子/段落级翻译。
- 校验器在每次缓存读写时执行，规则 0–2 成本可忽略，规则 3 对长文本采样后成本可控。
- 需要维护专有名词白名单和拒绝文本模式列表。
- 历史坏条目会在启动清理中被逐步淘汰；旧的内置热词条目也会因键格式/长度不符而被清理。

## 拒绝的替代方案

- **语义相似度校验**：引入向量/embedding 判断译文与源文相关性，准确性高但计算和存储成本过大，不符合扩展轻量定位。
- **完全禁用短文本缓存**：实现简单，但会损失大量 UI 短词（如 `"Login"`、`"搜索"`）的缓存收益。最终采用长度下限折中，而非完全禁用。
- **保留宽松归一化和内置热词库**：可维持较高命中率，但会继续产生近似命中。
- **仅写入时过滤**：无法处理升级前已存在的坏缓存条目。
