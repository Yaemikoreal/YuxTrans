# ADR 0003：轻量本地模型推荐与一键拉取

## 状态

已批准 / 实现中

## 背景

YuxTrans 浏览器扩展当前已支持通过本地 Ollama 服务调用开源模型，默认推荐模型为 `qwen3.5:0.8b`。然而，用户在设置页看到的状态信息仅是：

- Ollama 服务是否运行；
- 推荐模型是否已下载；
- 如果未下载，只能复制 `ollama pull` 命令到剪贴板，再手动打开终端执行。

这增加了普通用户的使用门槛。为了贯彻项目"轻量、准确、稳定"的核心目标，需要在不增加扩展体积、不引入额外依赖的前提下，让用户在设置页即可**自主选择并一键下载**经过验证的轻量开源模型。

## 决策

采用"方案 A：Ollama 生态内的轻量模型推荐 + 一键拉取"。

即：在扩展设置页内提供一个"轻量模型推荐"区域，列出若干体积小于 1GB、适合翻译场景的开源模型（默认以 `qwen3.5:0.8b` 为主推），用户点击"下载"后，扩展直接调用 Ollama 本地 API `/api/pull` 进行流式拉取，并在卡片上实时显示下载进度。

## 理由

| 评估维度 | 方案 A（Ollama + 一键拉取） | 方案 B（扩展内置 ONNX/WebGPU 模型） |
|---------|---------------------------|----------------------------------|
| 扩展体积 | 不增加扩展包体积 | 至少增加 100MB–500MB |
| 推理性能 | 依赖 Ollama，可选 CPU/GPU | 浏览器 WebGPU 兼容性差，CPU 慢 |
| 实现复杂度 | 低，仅增加 UI + fetch 流式进度 | 高，需引入 ONNX Runtime / transformers.js |
| 维护成本 | 低，模型列表可硬编码 | 高，需持续跟进浏览器 API 和模型格式 |
| 用户自主权 | 高，用户自主选择下载 | 中，通常默认打包 |
| 离线可用性 | 下载后离线可用 | 下载扩展后离线可用 |

方案 A 在"轻量"和"稳定"上明显更优，且与现有架构完全兼容。

## 推荐模型列表

首批推荐模型均面向翻译场景，体积控制在 1GB 以内：

| 模型名 | 约大小 | 优势 | 适用场景 |
|-------|-------|------|---------|
| `qwen3.5:0.8b` | ~600MB | 中文能力强，已验证 | 默认主推，中英互译 |
| `phi4-mini:latest` | ~2GB（备选）| 英文能力强 | 英译其他语言 |
| `gemma3:1b` | ~800MB | 多语言均衡 | 通用翻译 |

> 注：首批以 `qwen3.5:0.8b` 为主，其他模型作为可选。后续可根据实测效果扩展。

## 实现概要

### UI 位置

`extension/options.html` 的"本地模型 (Ollama)"区域内，新增一个"轻量模型推荐"卡片列表：

- 每个卡片展示：模型名、约大小、语言特点、当前状态（未下载 / 已就绪 / 下载中）。
- 操作按钮：下载 / 暂停 / 复制 pull 命令（降级）。
- 下载中的卡片显示细进度条和百分比。

### 核心逻辑

`extension/options.js` 新增 `pullModel(modelName, onProgress)`：

1. 调用 `POST http://localhost:11434/api/pull`，body 为 `{ name: modelName }`。
2. 读取 ReadableStream，按行解析 JSON，提取 `completed` / `total` 字段更新进度。
3. 完成后自动调用 `fetch /api/tags` 刷新本地模型列表。
4. 出错时显示错误提示，并提供"复制命令"降级。

### 权限

扩展 `manifest.json` 已声明 `http://localhost:*/*` 的 `host_permissions`，options 页面可直接请求 Ollama API，无需新增权限。

## 风险与兜底

- **下载耗时**：模型 500MB–1GB，需明确提示大小，并提供取消/复制命令降级。
- **磁盘空间不足**：Ollama `/api/pull` 会在空间不足时返回错误，前端捕获并提示。
- **Ollama 服务未启动**：先执行现有状态检测，未启动时禁用下载按钮。
- **网络中断**：流式接口会返回错误，前端重置状态并允许重试。
- **平台差异**：Windows 已有 `setup-ollama.bat`；macOS/Linux 用户可手动安装 Ollama，或后续补充 `setup-ollama.sh`。

## 后续可扩展

- 支持模型自动切换：当用户选择 local provider 但未下载任何模型时，自动弹出推荐下载向导。
- 支持远程模型列表：从 GitHub Release 或 CDN 拉取推荐模型配置，方便动态更新。
- 增加 `setup-ollama.sh` 覆盖 macOS/Linux。
